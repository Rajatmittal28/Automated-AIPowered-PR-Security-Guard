The Clever Part
Self-Reflection Loop
The AI second-guesses itself before blocking anyone's PR — the key to enterprise trust
01
Initial Finding
Mistral Codestral reads the diff and produces raw findings: type, location, severity, initial confidence score.
02
Adversarial Critique
A second AI call asks: "Is this actually in a test file? Is this an env variable reference? Could this already be sanitized upstream?"
03
Confidence Adjustment
Each finding gets a confidence score adjustment (−0.40 to +0.15). False positives are identified and discarded.
04
Threshold Gate
Only findings with final confidence ≥ 0.85 block the merge. Between 0.60–0.85 → warning. Below 0.60 → discarded silently.

Dependency Security
CVE Detection Engine
Real vulnerability database lookups — not AI guessing from memory
Step 1 — Fetch full pom.xml
When pom.xml appears in a PR, we fetch the COMPLETE file from GitHub (not just the diff). This catches pre-existing vulnerable dependencies, not just newly added ones.
Step 2 — Extract all dependencies
Parse every <dependency> block in the full pom.xml. Resolve property references like ${log4j2.version} back to real version numbers. Produces a list of groupId:artifactId:version tuples.
Step 3 — Query OSV database
Batch-query api.osv.dev for all dependencies at once. For each vulnerability ID found, fetch full details including CVSS score, CVE alias, and remediation link. Free, no API key, covers 50k+ packages.
ACCURACY COMPARISON
LLM ALONE
Only catches famous CVEs it saw in training. Log4Shell: yes. Newer 2024 CVEs: maybe. Confidence varies wildly.
OSV API + LLM
Every CVE in the database, regardless of fame. 0.98 confidence. CVSS score, fix version, compliance link — all factual.
